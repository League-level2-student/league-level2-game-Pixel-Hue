Chrome Dino 2

by: Gavin B

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I was already thinking about how I would make an endless runner and Chrome Dino was the first thing that came to my mind. Also, I didn't try to make a platformer game because I wasn't sure how I was going to create physics.

2. What I Learned

From user testing I learned I need to be more clear with my wording and you want the instructions to be the first thing you see.

3. Architecture of My Game

I have 6 classes, ChromeDino, Dino, GameObject, GamePanel, ObjectManager, and Obstacle. The ChromeDino class sets up the window/frame. The Dino and Obstacles class hold the image, x, and y variables so the GamePanel can tell the ObjectManager to draw the dino and cacti. The GamePanel class is what keeps track of what graphics are supposed to be seen and saying what to draw. While, the ObjectManager is what draws everything and checks collision.

4. Technical Implementation

I am proud that I was able get the cacti to spawn in a 64x grid and make it spawn the perfect amount so the game isn't impossible. When I was first making the game I realized it was really hard to dodge cacti with the movement the Dino had. So first, I came up with the idea to make the Dino move faster, but after a bit of testing it felt awkward to play with. After that, I then came up with the idea of limiting the movement with a grid so it would be easy to tell where you would end up. This turned out to work much better and I am proud of it because it made the game a lot more fun.

5. Most Challenging Part

The most challenging part about this project was loading images. I solved the not being able to load images problem after I had the idea it was the image itself that was causing the problem so I checked it and discovered that it was a jpg when I was trying to load a png. So I fixed the issue by using a png rather than a jpg.

6. Future Fixes and Updates

If I were to keep working on this game I would add a high-score system and make the Dino's movement less static.
